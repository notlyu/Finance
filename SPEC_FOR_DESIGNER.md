# Спецификация для дизайнера — Финансовое приложение

## 1. Общая информация

**Название:** Семейный финансовый трекер
**Описание:** Приложение для управления семейными и личными финансами несколькими пользователями
**Платформы:** Desktop (1024px+), Mobile (320px-768px), Tablet (768px-1024px)
**Язык интерфейса:** Русский
**Дизайн-система:** Material Design 3 (кастомная реализация на CSS-переменных + Tailwind)
**Поддержка:** Dark / Light mode (переключается в настройках)

---

## 6. Страницы — полное описание

### 6.1 /login — Вход в систему

**Цель:** Авторизация или регистрация нового пользователя

**Секции:**
1. **Фон:** Градиентные декоративные круги (фиолетовый, зелёный) по углам
2. **Центральная карточка:**
   - Логотип + название "Финансы"
   - Переключатель "Вход" / "Регистрация"
   - Email поле
   - Пароль поле (с иконкой показа/скрытия)
   - "Забыли пароль?" ссылка
   - Кнопка "Войти" / "Зарегистрироваться"
3. **Социальное доказательство:** Аватарки + "12k+ семей уже с нами"
4. **Блок ошибки:** Красный контейнер при неверных данных

**Состояния:**
- Default
- Loading (кнопка disabled + текст "Отправка...")
- Error (красный блок сверху)
- Success (редирект)

**Адаптив:** На мобильных — полноэкранная карточка без декора

---

### 6.2 /forgot-password — Восстановление пароля

**Цель:** Двухшаговый процесс сброса пароля

**Шаг 1 (request):**
- Поле Email
- Кнопка "Отправить ссылку"
- Ссылка "Вернуться ко входу"

**Шаг 2 (reset):**
- Поле "Новый пароль"
- Поле "Подтвердите пароль"
- Кнопка "Изменить пароль"

**Состояния:**
- Success: зелёный блок → автопереход на /login через 2 сек
- Error: красный блок

---

### 6.3 / — Главная (Dashboard)

**Цель:** Обзор финансового состояния семьи и личных финансов

**Секции (в порядке сверху вниз):**

1. **Блок предупреждений** (опционально)
   - Типы: "Нет свободных средств" (красный), "Цель почти достигнута" (жёлтый)
   - Иконка + текст + сумма

2. **Карточки баланса** (grid 2 колонки на desktop, stack на mobile)
   
   **Карточка "Семейные финансы"** (только если пользователь в семье):
   - Заголовок: "Семейные финансы"
   - Суммы: Баланс, Доход за месяц, Расход за месяц
   - Зарезервировано (цели + желания)
   - Свободные средства (gradient primary карточка)
   - Вклад участников (если > 1 члена):
     - Аватар + имя + сумма
     - Прогресс-бар (относительно среднего)
   
   **Карточка "Ваши финансы"** (всегда):
   - Заголовок: "Ваш личный вклад" (если в семье) / "Ваши финансы" (solo)
   - Суммы: Личный баланс, Доход, Расход
   - Кнопка "Создать семью" (если solo)

3. **Последние операции** (2/3 ширины на desktop)
   - Заголовок: "Последние операции" + ссылка "Все операции →"
   - Список: до 5-10 транзакций
     - Иконка категории (цветная circle)
     - Название категории (или 🔒 Сюрприз)
     - Имя автора (family) / ничего (solo)
     - Дата
     - Сумма (+зелёный / -красный)
   - Empty state: иконка receipt_long + "Нет операций"

4. **Активные цели** (1/3 ширины на desktop)
   - Заголовок: "Цели" + ссылка "Все →"
   - Карточки целей с прогресс-баром
   - Empty state: иконка track_changes + "Нет активных целей"

**Адаптив:**
- Desktop: 2 колонки для балансов, 3:1 для операции+цели
- Mobile: всё в stack, компактные карточки

---

### 6.4 /transactions — Операции

**Цель:** Полный список транзакций с фильтрами

**Секции:**

1. **Header:** Заголовок + счётчик за период + кнопка "Добавить"

2. **Фильтры** (collapsible на mobile)
   - Период: select (Сегодня / Вчера / Эта неделя / Этот месяц / Произвольный)
   - Тип: select (Все / Доходы / Расходы)
   - Категория: select со списком
   - Видимость: select (Все / Только видимые / Только скрытые)
   - Произвольные даты: 2 date picker (если выбран "Произвольный")
   - Кнопка сброса

3. **Таблица / список транзакций**
   
   Desktop — таблица:
   - Колонки: Дата | Категория | Сумма | Комментарий | Автор | Действия
   - Строки: зебра (чёт/нечёт)
   - Сумма: +зелёная / -красная
   - Действия: edit icon / delete icon (только для своих)
   - Скрытые: категория = 🔒 Сюрприз, сумма = •••••

   Mobile — карточки:
   - Иконка категории | Название | Дата | Автор
   - Сумма справа
   - Кнопки Изменить / Удалить

4. **Кнопка "Загрузить ещё"** (если есть ещё)

**Модальное окно "Добавить/Редактировать операцию":**
- Поле "Сумма" (крупное, number, ₽ справа)
- Segmented toggle: Расход | Доход
- Поле "Категория" (select, фильтруется по типу)
- Поле "Дата" (date picker)
- Поле "Комментарий" (textarea)
- Toggle "Скрыть от семьи"
- Кнопки: Отмена / Добавить

**Предупреждение:** При превышении бюджета — confirm dialog с деталями

### 6.5 /goals — Цели накоплений

**Цель:** Создание и отслеживание финансовых целей

**Секции:**

1. **Celebration overlay** (при достижении)
   - Полноэкранный overlay с анимацией
   - 🎉 "Цель достигнута!"
   - Звуковой эффект (опционально)

2. **Header:** Заголовок + кнопка "Архив" + кнопка "Добавить цель"

3. **Табы:** "Семейные (N)" | "Мои (N)"

4. **Сетка целей** (grid, responsive)
   - Карточка цели (rounded-3xl, тень):
     - Иконка категории ( определяется по названию)
     - Название цели
     - Срок (если указан)
     - Прогресс-бар с %%
     - "Накоплено: X ₽ из Y ₽"
     - Кнопки: "Пополнить" | "Прогноз"
     - Если достигнута: галочка + зелёный badge

5. **Архивные цели** (если toggle "Архив" активен):
   - Достигнутые цели с датой
   - Кнопка "Вернуть в активные"

**Модальное окно "Новая/Редактировать цель":**
- Поле "Название" (text)
- Поле "Целевая сумма" (number, ₽)
- Поле "Текущая сумма" (number, опционально, для старых целей)
- Поле "Срок" (date, опционально)
- Поле "Ставка %" (number, для расчёта прогноза)
- Toggle "Автозачисление"
  - При включении: select (% от дохода / фикс. сумма) + number
- Toggle "Семейная цель" (если пользователь в семье)
- Кнопки: Отмена / Создать

**Модальное окно "Пополнить":**
- Инфо: цель, накоплено, осталось
- Свободные средства (сколько доступно)
- Поле "Сумма"
- Быстрые кнопки: 10% | 25% | 50% | 75% | 90% | 100%
- Warning при риске уйти в минус
- Кнопки: Отмена / Пополнить / Всё равно пополнить

### 6.8 /safety-pillow — Подушка безопасности

**Цель:** Отслеживание финансовой подушки безопасности

**Секции:**

1. **Основная карточка:**
   - "Ликвидные средства": крупная сумма
   - "Цель: X мес × Y ₽/мес = Z ₽"
   - "Зарезервировано: ..." (если > 0)
   - Прогресс-бар (крупный)
   - "X% от цели"

2. **Рекомендация:**
   - Если не достигнуто: синий/фиолетовый блок
     - "Откладывайте N ₽/мес чтобы достичь подушки за M мес"
   - Если достигнуто: зелёный блок "Подушка безопасности уже сформирована!"

3. **Уровни защиты** (grid 3):
   - "Минимальная" (3 мес) — красный

## 8. Адаптивность

### Breakpoints

| Breakpoint | Ширина | Поведение |
|------------|--------|-----------|
| Mobile | < 768px | Bottom nav, stacked cards, collapsed sidebar |
| Tablet | 768px - 1023px | Partial sidebar, 2-column grids |
| Desktop | ≥ 1024px | Full sidebar, multi-column layouts |

### Паттерны адаптива

1. **Sidebar → Bottom nav:**
   - Desktop: aside w-64 fixed left
   - Mobile: nav fixed bottom с 4-5 кнопками

2. **Table → Cards:**
   - Desktop: `<table>` с колонками
   - Mobile: карточки с vertical layout

3. **Filters:**
   - Desktop: всегда visible панель
   - Mobile: collapse под кнопкой "Фильтры"

4. **Grid → Stack:**
   - Desktop: grid grid-cols-2, grid-cols-3, grid-cols-4
   - Mobile: всё в 1 колонку

## 12. Заметки для дизайнера

### Что уже реализовано хорошо:
- Material Design 3 токены
- Dark/Light mode
- Консистентная система spacing, border-radius, shadows
- Чёткое разделение доходов (зелёный) и расходов (красный)
- Иконки Material Symbols
- Адаптивная верстка

### Что можно улучшить:
- Skeleton loaders (вместо простого спиннера)
- Toast-уведомления для success/error действий
- Анимации page transitions
- Empty states с иллюстрациями
- Tooltips для неочевидных элементов
- Confirmation dialogs (вместо window.confirm)

### Что не нужно менять:
- Цветовая схема (фиолетовый primary)
- Структура навигации
- Общая архитектура Layout
- Система токенов Material Design 3
